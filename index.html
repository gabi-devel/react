Qué es?

Es una librería que nos sirve para desarrollar interfaces de usuario web o clientes web, completamente dinámicas, reactivas y asíncronas con el servidor (todo irá por Ajax). Es decir que vamos a desarrollar una web en el lado del cliente, un front-end. Y este front-end lo que va a estar haciendo es consultar a un back-end o directamente funcionar en el front-end de manera muy dinámica. Y todo es completamente reactivo, es decir, yo hago un cambio en una propiedad de un componente y automáticamente ese cambio que hago en esa propiedad se muestra de manera instantánea en la vista o en lo que el usuario está viendo. Es para hacer aplicaciones single page, aplicaciones SPA.

Instalación

  • Instalar Node.js

    • Entrar a la página de nodejs.org . La más recomendable a largo plazo es la LTS. Descargar archivo .msi

  • Actualizar npm

    • npm es el gestor de paquetes de Nodejs, que nos sirve para bajarnos las diferentes dependencias, librerías y paquetes de JavaScript.
    En git bash: npm install -g npm@latest hace una actualización global. Esto nos va a permitir instalar luego React.
    Hacemos npm --version

  • Borrar la caché de npm

    • npm cache clean --force Porque si no borramos esta caché es posible que se nos descarguen versiones más antiguas a la actual que haya de React, entonces es recoendable siempre tener la caché limpia.
    clear (cls en windows) limpia la consola

  • Create React App

    • https://github.com/facebook/create-react-app
    Con este paquete, este módulo de Node, es un intérprete de consola, de cli. Nos va a permitir instalar React muy fácilmente e incluirá webpack para la compilación y minificación del proyecto. Además incluye un Live Reload, para que cuando yo haga un cambio de código, automáticamente se me compile, la pantalla se refrezque. Nos va a proporcionar la estructura básica de un proyecto de React. Permitirá hacer paquetes para producción y un montón de cosas. Esto sería lo equivalente a Angular cli. En todos los frameworks siempre hay un paquete de este tipo, un intérprete de consola.
    Hacemos npm install -g create-react-app

Generar el proyecto

En git bash, dentro de la carpeta donde se va a crear el directorio: create-react-app aprendiendo-react (el nombre de directorio no puede contener letras mayúsculas). Luego se puede cambiar a Camel Case.
Si se quiere, agregarle alguna letra mayúscula al nombre del directorio que se creó.
Entrar al directorio que se creó. Ejecutar ahí: npm start para que se ejecute en el servidor local, me compile todo y yo pueda probarlo en el navegador. Dejar corriendo el npm start en git bash mientras estemos trabajando con el proyecto.
Si nos situamos fuera del directorio: en git bash dir nos muestra las carpetas que hay. Luego ponemos cd y el nombre de la carpeta a la cual queremos entrar.
En ese último directorio creado ponemos: npm start y esperamos a que nos cargue el servidor web local. En Windows 7 me dio error una parte que no se carga porque no está instalado PowerShell. En git el nombre del directorio se ve en minúscula porque así fue cargado en package.json, que no me permite poner Camel Case. En package.json tenemos todas las dependencias que trae por defecto.
Git me devuelve dónde me está corriendo el servidor local. Copio esa url (que se encuentra en la imagen de la derecha) y la pego en el navegador.

Descargar Git en su página. Eso es Git Bash.

Git Bash: Bash es un lenguaje de comandos y shell de Unix, y es el Shell predeterminado en Linux (Ubuntu, etc.) y OS X. En términos simples, el git que se ejecuta en la terminal de cualquier dispositivo Linux se conoce como git bash.
Git CMD: (Símbolo de línea de comandos) es el intérprete de línea de comandos en los sistemas operativos Windows. Una especie de equivalente al terminal malo en Linux, cuando instala git en Windows y está acostumbrado a usar la línea de comandos, uno usa cmd para ejecutar comandos git. No hay ninguna diferencia en lo que respecta a los comandos git cuando usa cualquiera de los anteriores. Preferiría el terminal sobre cmd cualquier día, pero si te gusta Windows por razones convencionales, he usado Cygwin para darme el terminal como si fuera Windows.
